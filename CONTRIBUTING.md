# Contributing

Thank you for your interest in improving The Software Architecture Handbook.

## Contribution types

Useful contributions include:

- factual corrections
- clearer explanations
- improved examples
- diagrams
- references
- typo fixes
- code corrections
- accessibility improvements

## Before opening a pull request

1. Read `STYLE_GUIDE.md`.
2. Read `WRITING_GUIDELINES.md`.
3. Keep terminology consistent.
4. Run Markdown checks.
5. Explain the reason for the change.

## Pull request expectations

A pull request should:

- address one coherent topic
- avoid unrelated formatting changes
- preserve numbering
- include references for factual claims
- update `SUMMARY.md` when adding chapters

## ADR and Principle numbering

Do not assign a new ADR or Principle number without checking the current register.

## Commit style

Use concise imperative commit titles.

Examples:

```text
Add service architecture anti-patterns
Fix broken glossary links
Clarify consistency trade-offs
```

## Editorial authority

The maintainer may restructure, rewrite, combine, or reject contributions to preserve the coherence of the book.
